# Words with Toddlers - iOS

Complete iOS implementation of Words with Toddlers built with Swift and SwiftUI.

## Quick Start (Recommended)

### Method 1: Create Xcode Project Manually (5 minutes)

1. **Open Xcode** (version 15 or later)

2. **File → New → Project**
   - Select **iOS** → **App**
   - Click **Next**

3. **Project Settings:**
   - Product Name: `WordsWithToddlers`
   - Team: (Select your team or leave blank)
   - Organization Identifier: `com.wecodefire` (or your own)
   - Interface: **SwiftUI**
   - Language: **Swift**
   - Click **Next**

4. **Save Location:**
   - Navigate to: `words-with-toddlers/ios/`
   - **Important:** Uncheck "Create Git repository"
   - Click **Create**

5. **Delete Default Files:**
   - In Project Navigator, delete:
     - `ContentView.swift` (if created)
     - `WordsWithToddlersApp.swift` (if created)

6. **Add Source Files:**
   - Right-click on `WordsWithToddlers` folder in Project Navigator
   - Select **Add Files to "WordsWithToddlers"...**
   - Navigate to `ios/WordsWithToddlers/WordsWithToddlers/`
   - **Select all folders:** `Models/`, `Views/`, `Services/`, `Utilities/`
   - **Important:** Uncheck "Copy items if needed"
   - Choose "Create groups"
   - Click **Add**
   - Repeat to add `WordsWithToddlersApp.swift` from `ios/WordsWithToddlers/WordsWithToddlers/`

7. **Add Resources:**
   - Right-click on `WordsWithToddlers` folder
   - Select **Add Files to "WordsWithToddlers"...**
   - Navigate to `ios/WordsWithToddlers/WordsWithToddlers/Resources/`
   - Select both `word_lists/` and `sounds/` folders
   - **Important:** Uncheck "Copy items if needed"
   - Choose **"Create folder references"** (folders should appear blue/yellow)
   - Click **Add**

8. **Configure Info.plist:**
   - In Project Navigator, click on project name (top)
   - Select target `WordsWithToddlers`
   - Go to **Info** tab
   - Click **+** next to any key
   - If prompted, use the existing Info.plist at `WordsWithToddlers/Info.plist`

9. **Build and Run:**
   - Select a simulator (e.g., iPhone 15)
   - Press **Cmd+R** or click ▶️ Play button
   - App should build and launch!

### Method 2: Using Command Line (Alternative)

```bash
cd ios
open -a Xcode .
```

Then follow steps 2-9 above.

## Features Implemented

✅ **Complete feature parity with desktop version:**

- **Adaptive Difficulty System**
  - Tracks last 10 attempts
  - Auto level-up at ≥80% accuracy
  - Auto level-down at <50% accuracy
  - 8 grade levels (Pre-K → 6th)

- **Two Game Modes**
  - Discovery: Free typing with word detection
  - Challenge: Guided word practice

- **Challenge Modes**
  - Visual: See the word and type it
  - Audio: Hear the word and type it
  - 3-strike word reveal in audio mode

- **1000+ Educational Words**
  - Dolch word lists (220 words + 95 nouns)
  - Fry word lists (1000 high-frequency words)
  - Organized by grade level

- **Audio & Visual Features**
  - Text-to-speech (AVSpeechSynthesizer)
  - Sound effects (AVAudioPlayer)
  - Celebration animations
  - Colorful letter display

- **Settings & Persistence**
  - Sound selection
  - Typewriter mode toggle
  - Session history (JSON files)
  - Config persistence (UserDefaults)

## Troubleshooting

### Resources not found

If word lists or sounds aren't loading:
1. In Project Navigator, verify `Resources/word_lists/` and `Resources/sounds/` appear as **blue/yellow folders** (folder references)
2. If they appear as **regular folders**, remove them and re-add using "Create folder references"

### Build errors

If you get Swift compilation errors:
1. Clean build folder: **Product → Clean Build Folder** (Cmd+Shift+K)
2. Quit and restart Xcode
3. Verify iOS Deployment Target is set to 17.0+

### Speech not working

1. Test on a physical device (simulator sometimes has TTS issues)
2. Check device volume
3. Verify microphone/speech permissions if needed

### Keyboard not appearing

- Press **Cmd+K** in simulator to toggle software keyboard
- Or: **I/O → Keyboard → Toggle Software Keyboard**

## Development

### Testing Adaptive Difficulty

- Get words wrong intentionally - should level down after 10 attempts with <50% accuracy
- Get words right - should level up after 10 attempts with ≥80% accuracy
- In audio mode, get same word wrong 3 times - should reveal the word

### Adding New Sounds

1. Add `.wav` file to `Resources/sounds/`
2. Update `availableSounds` array in `SettingsView.swift`

### Modifying Word Lists

Word lists are loaded from `Resources/word_lists/`:
- Edit `.txt` files directly
- One word per line
- Will be converted to lowercase automatically

## Building for Release

1. **Select "Any iOS Device (arm64)"** as build target
2. **Product → Archive**
3. Follow Xcode distribution workflow for:
   - TestFlight
   - App Store
   - Ad Hoc distribution

## Architecture

- **UI Framework:** SwiftUI
- **State Management:** `@Published` + `@ObservableObject`
- **Text-to-Speech:** AVSpeechSynthesizer
- **Audio Playback:** AVAudioPlayer
- **Data Persistence:** UserDefaults + FileManager + JSON
- **Pattern:** MVVM (Model-View-ViewModel)
